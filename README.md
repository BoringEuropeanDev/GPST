# 📈 GPST — Global Predictive Stock Terminal

> **⚠️ DISCLAIMER: This is not financial advice. All predictions are generated by an automated system for informational and educational purposes only. Past performance does not guarantee future results. Always consult a licensed financial advisor before making investment decisions.**

---

A full-stack, AI-powered stock intelligence terminal built entirely on **free and open data sources**. No paid APIs required. Features a dark trading-terminal UI, real-time market data, ML-based directional predictions, news sentiment analysis, macroeconomic context, and continuous model evaluation.


---

## ✨ Features

- 🌍 **80+ Global Stocks & ETFs** — US, International, Sector ETFs, Commodities-linked
- 📊 **Interactive Price Charts** — 1M / 3M / 6M / 1Y / 2Y / 5Y historical OHLCV
- 🤖 **AI Predictions** — Direction (UP/DOWN/NEUTRAL) with probability distributions
- 📰 **News Sentiment Analysis** — GDELT + Alpha Vantage news feeds
- 🏦 **Macroeconomic Context** — FRED (VIX, Yield Curve, Fed Funds Rate, CPI)
- 🏭 **Sector Performance** — Real-time sector ETF tracking (XLK, XLV, XLF, etc.)
- 🔁 **Continuous Learning** — Predictions are evaluated daily and accuracy tracked
- 🎨 **Custom Terminal UI** — Dark aesthetic, no templates, scanline effects, animations

## 🚀 Quick Start — Local Development

### Prerequisites

- **Node.js** v18+ → https://nodejs.org
- **Python** 3.11+ → https://python.org
- **Git** → https://git-scm.com

*(Optional for production: Docker + PostgreSQL)*

---

### Step 1: Clone the Repository

```bash
git clone https://github.com/YOUR_USERNAME/global-predictive-stock-terminal.git
cd global-predictive-stock-terminal
```

---

### Step 2: Set Up the Backend

```bash
cd backend

# Create and activate a virtual environment
python -m venv venv

# On macOS/Linux:
source venv/bin/activate

# On Windows:
venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Copy environment config
cp .env.example .env
```

**Edit `.env`** and add your API keys (all free — see [Free API Keys](#-getting-free-api-keys)):

```env
DATABASE_URL=sqlite+aiosqlite:///./stockterminal.db
ALPHA_VANTAGE_KEY=your_key_here
NEWS_API_KEY=your_key_here
FRED_API_KEY=your_key_here
```

**Start the backend:**

```bash
uvicorn main:app --reload --port 8000
```

The backend will auto-initialize the database and begin loading stock data. Visit http://localhost:8000/docs to see the interactive API documentation.

---

### Step 3: Set Up the Frontend

Open a **new terminal window**:

```bash
cd frontend

# Install dependencies
npm install

# Start the development server
npm start
```

Visit http://localhost:3000 — the app will proxy API calls to the backend automatically.

---

### Step 4: (Optional) Run with Docker Compose

If you have Docker installed, you can run the entire stack with one command:

```bash
# From the project root:
cp backend/.env.example backend/.env
# Edit backend/.env with your API keys, then:

docker-compose up --build
```

- Frontend: http://localhost:3000
- Backend API: http://localhost:8000
- API Docs: http://localhost:8000/docs
- PostgreSQL: localhost:5432

## 🌐 Deployment on Vercel + Railway (Recommended Free Option)

### Backend → Railway

[Railway](https://railway.app) offers a free tier with enough resources to run this backend.

1. Go to https://railway.app and sign in with GitHub
2. Click **"New Project"** → **"Deploy from GitHub repo"**
3. Select your repository and choose the `/backend` folder
4. Railway will auto-detect the Dockerfile
5. Go to **Variables** and add your environment variables:
   ```
   ALPHA_VANTAGE_KEY=your_key
   NEWS_API_KEY=your_key
   FRED_API_KEY=your_key
   ```
6. Add a **PostgreSQL** plugin from the Railway dashboard (free tier)
7. Railway auto-sets `DATABASE_URL` — you're done!
8. Note your backend URL: `https://your-app.railway.app`

### Frontend → Vercel

1. Go to https://vercel.com and sign in with GitHub
2. Click **"New Project"** → import your repository
3. Set **Root Directory** to `frontend`
4. Add environment variable:
   ```
   REACT_APP_API_URL=https://your-app.railway.app/api
   ```
5. Edit `frontend/vercel.json` and replace `your-backend-url.railway.app` with your Railway URL
6. Click **Deploy**

Your app will be live at `https://your-project.vercel.app` 🎉

## 🤖 How the Prediction Engine Works

The AI prediction system combines multiple data sources using a weighted scoring model:

### Signal Sources (Weights)
| Signal | Weight | Source |
|--------|--------|--------|
| Technical Analysis | 40% | Yahoo Finance historical data |
| News Sentiment | 30% | GDELT + Alpha Vantage |
| Macroeconomics | 20% | FRED (VIX, Yield Curve, Fed Funds Rate) |
| Sector Momentum | 10% | Sector ETF performance |

### Technical Indicators Used
- **RSI** (Relative Strength Index) — overbought/oversold signals
- **MACD** (Moving Average Convergence/Divergence)
- **Simple Moving Averages** (5, 20, 50-day)
- **Bollinger Bands** — volatility and price position
- **Price Momentum** (1-day, 5-day, 20-day returns)
- **Volume Analysis** — volume ratios vs. averages

### Macro Signals
- **VIX** — Fear index (low VIX = bullish market environment)
- **10Y-2Y Yield Spread** — Yield curve inversion = recession warning
- **Federal Funds Rate** — High rates = headwind for equities

### Continuous Learning
Every day at 6 PM ET, the system:
1. Fetches actual closing prices for predictions made yesterday
2. Compares predicted vs. actual direction
3. Records accuracy in the `model_metrics` table
4. Logs error patterns for future model improvement

### Output Format
Each prediction includes:
- **Direction**: UP / DOWN / NEUTRAL
- **Confidence**: 50–95% (never artificially high)
- **Probability Distribution**: % bullish / neutral / bearish
- **Full Rationale**: Explanation of every signal
- **Sources List**: Every data source used
- **Disclaimer**: Mandatory legal disclaimer

## 🔧 Configuration

### Environment Variables

| Variable | Required | Description | Where to Get |
|----------|----------|-------------|--------------|
| `DATABASE_URL` | Yes | PostgreSQL or SQLite URL | Auto-set by cloud providers |
| `ALPHA_VANTAGE_KEY` | Optional | News sentiment + price data | alphavantage.co |
| `NEWS_API_KEY` | Optional | NewsAPI headlines | newsapi.org |
| `FRED_API_KEY` | Optional | Economic indicators | fred.stlouisfed.org |
| `MEDIASTACK_KEY` | Optional | Additional news | mediastack.com |

> Without any API keys, the system uses Yahoo Finance and GDELT (both keyless) and still functions fully.

### Adding More Tickers

Edit `backend/app/services/data_ingestion.py`, the `GLOBAL_TICKERS` dict:

```python
GLOBAL_TICKERS = {
    "AAPL": "Apple Inc.",
    "MSFT": "Microsoft Corporation",
    # Add your tickers here:
    "YOUR_TICKER": "Your Company Name",
}
```
